Welcome to Help for Balthisar Tidy!

Most of the options should be self-explanatory. If not, don't worry at this point; Balthisar Tidy is still very experimental, and documentation will be forthcoming.

Release History:

version 0.60
Added the following features:

  • Batch mode!

Implemented the following architectural changes:

  • Use TidyLib's native load/save routines to prevent the Mac OS from trying to do the conversions by itself. This actually works transparently to the user most of the time, but gave unexpected results because TidyLib tried to convert the encoding formats, too.
  • For display purposes, synched the text-view encodings manually to accomodate the user-selected input- and output-encodings.

Corrected the following bugs from 0.50:

  • fixed a drawing problem when clicking on an error and being transported to that error.
  • fixed a problem whereby clicking an error, moving the text view, and then clicking the error again wouldn't reshow the error in the HTML viewer/editor.

version 0.50
Complete rewrite of the entire application, encompassing the following:

  • Rewrote the Cocoa wrapper to TidyLib to make is more Cocoa-like.
    • Behavior is more like what I think a Cocoa programmer expects.
    • Dynamically generates all preferences based on compiled-in TidyLib.
  • Implemented virtually all Tidy options.
  • Upgraded to the 01-Mar-03 version of TidyLib.
  • The multiple options required a table-like interface to the options settings. Please tell me if it's too Windows-like (personally I like having the large quantity of options in a scroller).
  • Added a new mechanism to add line numbers to NSTextView classes.
    • Added line numbering to the Source HTML viewer/editor.
    • Added line numbering to the Tidy'd HTML viewer.
  • Added a new mechanism to turn on temporary highlighting in NSTextView classes.
    • When user selects an error, the Source HTML viewer/editor highlights and moves the user to the location of that error.
  • Replaced the TidyLib text output viewer with a more Mac-like table view of errors.
    • No Unix text (but no Tidy Information, either).
    • Selected a line transports the user to the error in the HTML viewer/editor.
